# 浏览器代理一键切换工具 - 功能文档

## 1. 项目概述
这是一个为 Windows 用户设计的极简、高效的浏览器/系统代理切换工具。它通过修改系统注册表实现代理的快速切换，并支持通过 JSON 文件灵活配置多个代理方案。

## 2. 核心功能
- **多代理方案管理**：支持预设多个 HTTP/SOCKS 代理配置。
- **一键快速切换**：双击运行脚本，输入编号即可完成切换，无需进入复杂的系统设置。
- **捕获当前配置 (s 键)**：能够读取当前系统中手动设置的代理（服务器地址和排除列表），并将其保存为新的配置项，方便后续一键恢复。
- **独立排除列表 (Bypass)**：支持为每个代理配置独立的排除列表（不使用代理的地址）。
- **即时刷新机制**：通过调用系统 `wininet.dll` 接口，使代理变更在 Chrome、Edge 等现代浏览器中立即生效，无需重启浏览器。
- **快捷编辑模式 (e 键)**：在工具界面直接输入 `e` 即可调用记事本编辑配置文件。

## 3. 文件结构说明
- **proxy-switcher.bat**：主入口程序。这是一个 ASCII 编码的批处理包装器，负责启动 PowerShell 脚本并处理潜在的编码冲突。
- **proxy-switcher.ps1**：核心逻辑脚本。使用 PowerShell 编写，负责 UI 显示、JSON 解析、注册表操作以及系统接口调用。支持完整的 UTF-8 中文显示。
- **config.json**：配置文件。存储所有的代理方案和全局默认设置。

## 4. 配置指南 (config.json)
```json
{
  "proxies": [
    {
      "name": "配置名称",
      "server": "ip:port 或 socks=ip:port",
      "enable": true,
      "bypass": "排除的主机名或 IP"
    }
  ],
  "default_bypass": "全局默认排除列表"
}
```
- `name`: 方案在菜单中显示的名称。
- `server`: 代理服务器地址（例如 `127.0.0.1:8888`）。为空时表示不使用代理服务器。
- `enable`: `true` 开启代理服务器，`false` 关闭代理服务器。
- `bypass`: 该方案独有的排除列表（分号分隔）。如果为空，则可回退到默认设置。

## 5. 使用方法
1. **启动**：双击 `proxy-switcher.bat`。
2. **选择**：根据屏幕显示的编号，输入对应的数字并按回车。
3. **捕获**：如果您在系统设置中调整了好的代理，返回工具输入 `s` 并命名，即可将其永久保存。
4. **管理**：输入 `e` 可以直接修改 `config.json` 进行批量维护。

## 6. 注意事项
- **系统权限**：本工具修改的是当前用户（HKCU）的注册表，通常不需要管理员权限。
- **杀毒软件**：由于工具涉及注册表修改，部分杀毒软件可能会弹出提醒，请选择允许运行。
- **浏览器适配**：支持所有默认遵循系统代理设置的浏览器（Chrome, Edge, IE, 微信内置浏览器等）。部分 Firefox 版本可能需要手动在浏览器内设置为“使用系统代理设置”。
