# 浏览器代理切换工具 详细设计说明

## 模型与数据库设计 (Model & Database Design)
本项目使用 JSON 文件作为配置存储。

### 核心配置文件 (`config.json`)
| 字段名 | 类型 | 说明 | 备注 |
| :--- | :--- | :--- | :--- |
| proxies | Array | 存储代理模板的列表 | 每个元素包含 name, server, enable, bypass |
| default_bypass | String | 默认的直连列表 | 如果配置项未定义 bypass 时使用 |

## 变更记录 (Change Logic)
### v1.1.0 - 路径逻辑修复
- **修改前**: `$ConfigFile = "config.json"`。
- **修改后**: `$ConfigFile = Join-Path $PSScriptRoot "config.json"`。

### v1.2.0 - 状态检测与高亮
- 从注册表实时读取系统当前代理状态并显示在列表中（Red 高亮）。

### v1.3.0 - 交互式导航与重排
- **主循环升级**: 将 `Read-Host` 的输入驱动模式改为 `[Console]::ReadKey` 的按键驱动模式。
- **光标逻辑**: 引入 `$cursorIdx` 控制选中项，渲染时通过 `BackgroundColor` 进行高亮。
- **物理排序**: 通过 `U`/`D` 键直接操作内存中的代理数组元素索引并写回文件，实现列表物理顺序持久化。
- **配置管理**: 实现 `Delete` 快捷删除配置项。
- **稳定性**: 强制在 `$()` 字符串内对非 ASCII 字符的处理方案进行优化（UTF-8 with BOM）。

## 设计决策与原因 (Rationales)
1. **ReadKey 机制**: 它是原生 PowerShell CLI 实现复杂 UI 的关键，比 `Read-Host` 更具现代感。
2. **UTF-8 with BOM 强制**: 为了兼容 Windows 10/11 的默认控制台，必须包含 BOM 否则中文脚本执行时会报错或乱码。
